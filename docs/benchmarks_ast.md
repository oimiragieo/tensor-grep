# AST Benchmarks

Native Rust AST structural search using embedded `ast-grep-core` crates,
benchmarked against the official `sg` (ast-grep) CLI.

## Current Accepted Line

Corpus: 1000 Python files, 50000 LOC (deterministic, generated by `gen_corpus.py`).
Pattern: `def $F($$$ARGS): return $EXPR`

```
tg run   median: 325ms  (rayon parallel, ignore crate, lazy line_starts)
sg run   median: 444ms
ratio:   0.73 (tg is 1.37x faster)
```

Measured via `hyperfine --runs 15` on Windows amd64.

## Parity

40/40 structural match parity cases pass across Python, JavaScript,
TypeScript, and Rust. Validated by `run_ast_parity_check.py`.

## History

| Date | tg | sg | Ratio | Change |
|------|----|----|-------|--------|
| pre-parallel | 929ms | 311ms | 2.98x slower | Sequential walkdir, read_to_string |
| rayon+ignore | 350ms | 354ms | 0.99 (parity) | Par_iter, ignore crate |
| +lazy line_starts | 325ms | 444ms | 0.73 (1.37x faster) | Deferred line_starts, fs::read, no sort |

## How to Run

```powershell
python benchmarks/run_ast_benchmarks.py --output artifacts/bench_run_ast_benchmarks.json
python benchmarks/run_ast_parity_check.py --output artifacts/ast_parity_report.json
```
