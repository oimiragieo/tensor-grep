## The root cause (the design corrected the audit's framing)
The audit called it "95% payload filler." The mechanism is worse than a missing cap:
- **`defs`/`refs`/`source` don't compute "related" anything.** `build_symbol_defs_from_map` (`repo_map.py:13336-13479`) does `payload = dict(repo_map)` (`:13342`, shallow-copy the WHOLE whole-repo scan) then `payload["tests"] = list(repo_map.get("tests", []))` (`:13346`) — the repo's ENTIRE test-file inventory (`build_repo_map` `:6010/6037-6038`, every `_is_test_file` up to the 2000 scan cap), zero symbol-relevance filtering, then **duplicated a second time** into `related_paths` (`:13397-13400`). `refs` (`:13943-13950`) + `source` (`:13567`) inherit the identical dump. THIS is the "69KB for a 1-symbol answer / 344 of 350 entries."
- **`callers`/`impact` DO compute relevance** via `_relevant_tests_for_symbol` (`:3539-3675`, import-graph reverse-distance + personalized PageRank) but that fn has **no count cap** (only a deadline early-break) → `payload["tests"] = related_tests` uncapped (callers `:14954`, impact `:13793`). THIS is the "200KB / 464-entry callers array."
- **`blast-radius` is the ONE command already fixed:** `_apply_blast_radius_output_limits` (`:15113-15236`) caps `tests` (`:15158`) + stamps an `output_limit` block (`total/returned/omitted`, `:15209-15235`), wired on-by-default (`_DEFAULT_BLAST_RADIUS_JSON_MAX_FILES=25`, `main.py:68`). **This is the precedent to extract+generalize, not a pattern to invent.**

## The answer-first template to reuse (do NOT invent)
`build_agent_capsule` (`agent_capsule.py:1824-2302`) already ships the envelope: `omissions:{token_budget, omitted_section_count, omitted_sections[], follow_up_reads[]}` (`:2262-2267`), built by `_build_snippets` (`:1395-1437`, appends a visible omission record instead of silent-drop) + `_follow_up_reads` (`:1440-1485`) + `_command_ref` (`:898-903`, a literal copy-pasteable follow-up argv). Omissions feed trust: `_capsule_context_consistency` (`:1488-1520`) downgrades confidence if the PRIMARY target was omitted. **One genuine delta from the template:** here the natural follow-up pointer is SELF-referential ("re-run with a bigger `--max-tests`/`--max-tokens`"), vs the capsule pointing at a DIFFERENT command → gets its own explicit test, not assumed-correct-by-analogy.

## `--max-tokens` — 4 divergent mechanisms + 1 no-op + total absence (worse than the audit's "3")
(a) `orient` snippet-list-only (`main.py:7372`→`orient_capsule.py:236-263`; the reported `token_estimate` covers the whole doc but the budget only bounded snippets — can exceed its own advertised budget). (b) `agent` source-block-only (`:7741`→`_apply_source_output_budget` `repo_map.py:12420-12425`). (c) `context-render` mechanically identical to (b) despite whole-bundle help text (default 16000). (d) `context` (plain) = the ONLY true whole-payload budget: `_apply_context_token_budget` (`repo_map.py:7362-7399`) serializes the whole dict + iteratively shrinks `max_files` (default 16000, min=0, 0=unbounded). (e) `edit-plan` = a documented no-op (honest). (f) **defs/refs/callers/impact/blast-radius = `--max-tokens` ABSENT entirely** (only `--max-repo-files`/`--deadline`, + count-caps on blast-radius). **Audit citation correction:** `main.py:7521-7529` is the context-RENDER daemon RPC (mechanism c), NOT the plain-context whole-pack; the real whole-payload logic is `repo_map.py:7362-7399`.

## Design
1. **Fix the root cause BEFORE any budget** (else the cap just hides the bug at a smaller size): (a) swap `repo_map.py:13346` raw-copy → `_relevant_tests_for_symbol` (the fn callers/impact already use) so defs/refs/source stop dumping the whole manifest; (b) rebuild `related_paths` from the NOW-filtered `tests` in all 3 sites (`:13397-13400` defs, `:14935-14943` callers, `:13773-13776` impact) — apply the filter/cap BEFORE `related_paths` derives or the bytes leak back via the second field.
2. **Bound the (now-relevant) tests** — extract `_apply_blast_radius_output_limits`'s tests-cap + `output_limit` stamping into a shared, `field_name`-generic helper; give defs/refs/callers/impact a DEDICATED `--max-tests` (blast-radius conflates it with `--max-files` today).
3. **Answer-first `omissions` envelope** (additive sibling key — do NOT restructure `tests`/`definitions`/`callers` themselves, §contract) with `token_budget`/`tests_cap`/`omitted_sections`/self-referential `follow_up_reads`.
4. **Universal `--max-tokens`** modeled on mechanism (d)'s serialize-then-measure, but answer-first SHRINK ORDER: trim SECONDARY fields first (`tests`/`related_paths`/`import_graph_consumers`), only then the PRIMARY answer array (`definitions`/`references`/`callers`) — and when the primary is cut, say so DISTINCTLY (an agent trusting "here are all N callers" must know N was cut). Default 16000, min=0 (match `context`, not orient/agent's 1200-3000 snippet defaults).

## Contract safety (the load-bearing distinction)
The scan-truncation-vs-token-trim line is ALREADY a tested contract (`_scan_incomplete` docstring `main.py:8447-8465`; blast-radius comment `:9260-9273`): an OUTPUT cap = COMPLETE analysis capped for display = **exit 0**; only `scan_limit`/`caller_scan_limit`/`partial`/`caller_scan_truncated` = **exit 2**. The new `--max-tokens`/`--max-tests` trim MUST land on the exit-0 side (new `omissions`/`output_limit` block) and MUST NEVER set `result_incomplete`/`partial`/`caller_scan_limit`. `JSON_OUTPUT_VERSION` (`repo_map.py:141`) stays 1 — `omissions` is additive; the project shipped result_incomplete + native-delegation as additive-at-v1 (precedent). **No golden risk** (`test_output_golden_contract.py`/`test_output_snapshots.py` are 100% `tg search` cases). **Real regression surface = `test_cli_modes.py`** (pins `payload["tests"]` as a FLAT ordered list of path strings, `:4048/4128/4166/4453/4584/4709/4786/7765`; all fixtures have ≤2 test files so a generous default passes them — but do NOT restructure `tests` into an object; add the sibling key, mirroring the already-tested `context --max-files` `output_limit` shape at `:4167-4173`). **MCP shares the builders** (`mcp_server.py:46-51` imports build_symbol_* directly) → the fix reaches MCP free, but sweep `test_mcp_server.py`/`test_mcp_caps.py` for tests-size assumptions.
