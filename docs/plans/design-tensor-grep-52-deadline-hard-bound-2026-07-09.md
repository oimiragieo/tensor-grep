## Root cause (one sentence)
"Each stage bounded" was never re-verified after task #61 shipped — two *new* per-file loops (B, D) and one *infrastructure* loop (A) sit outside every deadline check, and one builder (C, `impact`) never received deadline plumbing at all, so a command finishes its bounded stages inside budget yet blows the deadline in a loop nobody watches. Mechanistic fit for the receipt `tg callers ... main --deadline 15` = 354s/23x.

## The four unbounded loops (file:line)
- **A — `_iter_repo_files` file-tree walk** (`repo_map.py:943-1010`, bucket-interleave `:986-1006`). No deadline param; bounded only by `max_files` COUNT. `build_repo_map` calls it (`:5793-5799`) BEFORE its own parse-loop deadline check starts (`:5824-5829`). `build_inventory` is starker: `inventory.py:199` walks BEFORE `inventory.py:221` even creates the deadline var → the walk is 100% unbounded (explains `inventory --deadline 30` = 76s; inventory's own per-file loop `:224-227` IS bounded). WSL2 cross-fs `stat()` overhead amplifies.
- **B — `_relevant_tests_for_symbol`** (`repo_map.py:3510-3629`). No deadline params. Two unguarded `any()` loops over the FULL `tests` list (`:3584-3597` direct-definition, `:3608-3624` related), each a nested `tests × candidates × definitions` cross-product. **Dominant cause of the `main` repro:** `_preferred_definition_files` (`:3468-3507`) returns the full unfiltered `definition_files` when every import-score is 0 (`return preferred or definition_files`, `:3507`) — true for `main` (many defs, ~0 import edges) → floods B with zero bound. Deadline dropped at both call sites (`:14808-14815` callers, `:13616-13622` impact).
- **C — `build_symbol_impact_from_map` has NO deadline plumbing** (`repo_map.py:13554-13560`). Its `_preferred_definition_files` call (`:13594`) passes no deadline, defeating that function's own #61 fix. Acknowledged in-code: `_DeadlineBreakFlag` docstring (`:627-638`) names this function a "non-deadline-aware seam"; `test_repo_map_deadline.py:401-402` calls it out verbatim. `build_symbol_blast_radius_from_map` inherits via `:15174-15178`.
- **D — `string_refs` second pass in `build_symbol_refs_from_map`** (`repo_map.py:14035-14040`). Unbounded `for current in bounded_files: string_refs.extend(_string_literal_references(...))` after the properly-bounded main scan (`:13877-13884`). `_string_literal_references` (`:13778-13817`) is uncached `read_text` (`:13790`) + per-call `re.compile` (`:13797-13799`) + O(N) `count("\n")` (`:13802`), across up to 512 files.

2.
