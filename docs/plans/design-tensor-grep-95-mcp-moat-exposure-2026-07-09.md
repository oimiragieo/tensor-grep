## GATE VERDICT + MUST-FIX CORRECTIONS (Opus design-gate 2026-07-09, agent ade28b84) — VERDICT: SOUND-WITH-CHANGES
The threat model is real (verified: primary `path=` unconfined on tg_search/repo_map/agent_capsule/all symbol/session/checkpoint/rewrite tools — an arbitrary-dir READ, and WRITE on rewrite_apply/checkpoint), the `_confine_read_path` mechanism fails closed (`.resolve()` follows symlinks + rejects non-descendants), and the `--config` deferral is CORRECT + load-bearing (`_iter_yaml_files` does an UNCONFINED `(base_dir/rel_dir).resolve().rglob` — an absolute/`../` rel_dir escapes; confining only the top-level `config=` is insufficient). **3 MUST-FIX before building:**
1. **The ratchet test must be FAIL-CLOSED on UNCLASSIFIED string params, not name-matched.** "Probe each path/file/root param" lets a future tool with a param named `directory`/`target`/`src`/`out_dir` escape (the exact one-at-a-time recurrence). Instead: enumerate EVERY `str`/`str|None` param of every tool from `tool.inputSchema.properties`; assert each is EITHER in an explicit `CONFINEMENT_EXEMPT` allowlist (pattern, query, symbol, session_id, lang, ruleset, replacement, glob, type_filter, language, justification, model, provider, inline_rules, signing_key, checkpoint_id, …) OR rejects an out-of-root probe with `invalid_input`. A new unclassified param then FAILS the test until consciously confined/listed. This is the real ratchet vs a false one.
2. **Make the ratchet test BIDIRECTIONALLY valid** (oracle discipline): supply VALID in-root values for all co-required params so the tool actually REACHES the confinement check (else it false-passes on an unrelated early error, e.g. missing symbol/session_id); assert the rejection is specifically the confinement `invalid_input`; AND assert a valid in-root path is ACCEPTED. Without this the test is green while confinement is absent.
3. **Insert `path = str(_confine_mcp_path(path))` at the TOP of each tool, BEFORE any secondary anchor derives from `path`.** Two tools derive a confinement anchor from the raw primary path — `tg_ruleset_scan` (`scan_root`) and `tg_session_file_importers` (`session_root = Path(path).resolve()`). The latter is a LIVE VULN today (set `path=/etc`, its `file` confines to `/etc` → read `/etc/*`); confining `path` atop the function fixes it by NESTING — do NOT re-anchor `file` to cwd (breaks a legit relative `file` when session_root ≠ cwd).
**SHOULD-FIX before merge:** bump `_TG_MCP_SERVER_CONTRACT_VERSION` 1.0.0→1.1.0 + changelog + update the pin `test_mcp_server.py:1614-1615` (confinement IS a breaking behavior change for out-of-cwd callers — the design under-called this); bound `inline_rules` length (SafeLoader doesn't stop a YAML expansion-bomb DoS); `_mcp_root()` must treat unset/empty TG_MCP_ROOT as cwd (NOT `""`→filesystem-root) + validate the override is a real dir. **Build order confirmed: confinement-first; land the CORRECTED ratchet test (fixes 1+2) as step 3 so it GATES the bulk rollout (step 4).** Coverage rides the enumeration, NOT the prose tool-list (the prose undercounts — round-6 single-file tools already confine their paths; several unconfined scan tools like tg_ast_search/tg_index_search/tg_audit_history/tg_context_pack are omitted from the prose but caught by the ratchet).
